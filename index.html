FAQ

Questions answered.


Is this real NASA data or a simulation? +
The bundled nasa_data/ folder contains the actual train_FD001.txt, test_FD001.txt, and RUL_FD001.txt files from NASA's official Prognostics Data Repository (Saxena & Goebel, 2008). These are real turbofan engine telemetry recordings — not synthetic data. A synthetic simulation mode also exists for fast experimentation, clearly labeled as such in the UI.
Can I connect it to my real plant SCADA system? +
Not in the open-source version — it reads CSV/Excel files or the bundled NASA data. The consulting package adds OPC-UA and Modbus live data connectors to link directly to your PLC or DCS historian.
Is this scientifically valid for a research paper or master's thesis? +
The evaluation is methodologically sound: genuine 70/30 train/test split with no data leakage, F1/Precision/Recall/ROC-AUC on held-out data, RUL comparison against NASA's published ground truth. The RUL model uses the standard piecewise-linear approach from the C-MAPSS literature (Heimes 2008). For a journal submission you would also need multi-unit cross-validation across all 100 FD001 test engines — that's the next roadmap item.
What Python version and hardware do I need? +
Python 3.9–3.11 (not 3.12+ yet). Works on any x86 Windows/Mac/Linux machine. The Autoencoder training on 1000 samples takes about 30–60 seconds on a regular laptop CPU — no GPU required. The fault localization step (leave-one-out) takes 2–5 minutes; reduce epochs for faster iteration.
Why Streamlit instead of a web app? +
Streamlit makes the full ML pipeline interactive without a separate frontend/backend split — ideal for a research tool you run locally on sensitive industrial data that never leaves your machine. A cloud SaaS version is possible in a future version.
